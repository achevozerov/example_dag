## Краткое пояснение к коду

Для демонстрации выгрузил два файла: example_dag.py содержащий непосредственно настройки DAG'а, и source/amplitude.py с основным его функционалом.

### example_dag.py

DAG исполняется 4 раза в день каждые 6 часов и состоит из двух структурных частей (тут они реализованы через TaskGroup):
1) rdl (Raw Data Level) - загрузка данных в сырой слой, эти таски все ссылаются к файлу source/amplitude.py
2) cdl (Core Data Level) - здесь таски обращаются к dbt для запуска моделей которые выгружают данные в ядро хранилища

Кроме них используются также функции из файла source.alerting (его не добавлял), эти функции отвечают за отправку метрик в Zabbix для мониторинга работы дага и оповещения если что-то сломается

### amplitude.py

Здесь основная бизнес-логика находится внутри класса Amplitude_etl_cls где метод download отвечает за выгрузку данных на диск, а метод upload за отправку их в Clickhouse.
Обе эти задачи исполняются в функции-обёртке которая менеджерит процесс и вызывается внутри таски Airflow
